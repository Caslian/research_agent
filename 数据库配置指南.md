# InnoCore AI 数据库配置指南

> 本文档详细说明项目所需数据库的类型、用途、配置方法及快速启动方案。

---

## 📊 数据库总览

| 数据库 | 类型 | 用途 | 连接方式 | 状态 |
|--------|------|------|----------|------|
| **PostgreSQL** | 关系型数据库 | 用户、论文、分析报告、引用缓存等结构化数据 | `asyncpg` (异步驱动) | ⚠️ 需配置 |
| **Qdrant** | 向量数据库 | 论文内容的向量嵌入存储与语义检索 | `qdrant_client` | ⚠️ 需配置 |
| **Redis** | 键值缓存数据库 | 预留（当前代码中未主动调用） | `redis` (Python客户端) | 可选 |

## 1. PostgreSQL 配置

### 1.1 用途说明

PostgreSQL 是 InnoCore AI 的**主关系数据库**，通过 `asyncpg` 异步驱动访问。

**存储的数据表：**

| 表名 | 用途 |
|------|------|
| `users` | 用户账户信息 |
| `papers` | 论文元数据（标题、作者、摘要、DOI等） |
| `user_paper_relations` | 用户与论文的关联关系（收藏、评分等） |
| `analysis_reports` | 论文分析报告（摘要、创新点、局限性等） |
| `reference_cache` | 引用缓存（DOI → BibTeX） |

**核心字段示例：**
```sql
-- 用户表
users(id UUID, email VARCHAR(255), profile JSONB, created_at TIMESTAMP)

-- 论文表
papers(id UUID, title TEXT, authors TEXT[], abstract TEXT, doi VARCHAR(255), 
       file_path TEXT, content_hash VARCHAR(64), is_preset BOOLEAN, created_at TIMESTAMP)

-- 分析报告表
analysis_reports(id UUID, paper_id UUID, summary TEXT, innovation_point TEXT, 
                 limitation TEXT, future_idea TEXT, vector_ids JSONB, created_at TIMESTAMP)
```

### 1.2 环境变量配置

在 `.env` 文件中添加以下配置：

```env
# PostgreSQL 数据库配置
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_NAME=innocore_ai
DATABASE_USER=postgres
DATABASE_PASSWORD=your_password_here
```

对应的配置类定义在 `core/config.py` 的 `DatabaseConfig`：

```python
@dataclass
class DatabaseConfig:
    """关系数据库配置"""
    host: str = "localhost"
    port: int = 5432
    database: str = "innocore_ai"
    username: str = "postgres"
    password: str = "password"
    pool_size: int = 10
```

### 1.3 快速启动 PostgreSQL（Docker）

```bash
# 方式一：Docker 快速启动
docker run -d \
  --name innocore-postgres \
  -e POSTGRES_DB=innocore_ai \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=your_password_here \
  -p 5432:5432 \
  postgres:16-alpine

# 方式二：使用 Docker Compose（推荐）
# 在项目根目录创建 docker-compose.yml：
```

**docker-compose.yml 示例：**
```yaml
version: '3.8'
services:
  postgres:
    image: postgres:16-alpine
    container_name: innocore-postgres
    environment:
      POSTGRES_DB: innocore_ai
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: your_password_here
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

启动命令：`docker-compose up -d`

## 2. Qdrant 向量数据库配置

### 2.1 用途说明

Qdrant 是**向量数据库**，用于存储论文内容的向量嵌入，支持语义搜索。

**向量集合：**

| 集合名 | 类型 | 用途 |
|--------|------|------|
| `innocore_l1_preset` | L1 预置库 | 系统预设的论文向量（全局知识库） |
| `innocore_l2_user` | L2 用户库 | 用户上传/收藏的论文向量（个性化） |

**向量维度：** 1536（与 OpenAI `text-embedding-3-small` 兼容）

**核心功能：**
- `add_to_l1()` - 添加论文到预置库
- `add_to_l2()` - 添加论文到用户库
- `hybrid_search()` - 混合搜索（向量 + 关键词）

### 2.2 环境变量配置

在 `.env` 文件中添加以下配置：

```env
# Qdrant 向量数据库配置
QDRANT_HOST=localhost
QDRANT_PORT=6333
QDRANT_API_KEY=  # 本地部署可不填，云服务需要

# Embedding 模型配置（用于生成向量）
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_BASE_URL=https://api.openai.com/v1
```

对应的配置类定义在 `core/config.py` 的 `VectorDBConfig`：

```python
@dataclass
class VectorDBConfig:
    """向量数据库配置"""
    db_type: VectorDBType = VectorDBType.QDRANT
    host: str = "localhost"
    port: int = 6333
    api_key: Optional[str] = None
    collection_name_prefix: str = "innocore"
    embedding_model: str = "text-embedding-3-small"
    embedding_base_url: Optional[str] = None
```

### 2.3 快速启动 Qdrant（Docker）

```bash
# 方式一：Docker 快速启动（推荐）
docker run -d \
  --name innocore-qdrant \
  -p 6333:6333 \
  -p 6334:6334 \
  qdrant/qdrant:latest

# 方式二：使用 Docker Compose（推荐）
```

**docker-compose.yml 示例：**
```yaml
version: '3.8'
services:
  qdrant:
    image: qdrant/qdrant:latest
    container_name: innocore-qdrant
    ports:
      - "6333:6333"
      - "6334:6334"
    volumes:
      - qdrant_data:/qdrant/storage

volumes:
  qdrant_data:
```

启动命令：`docker-compose up -d`

**验证 Qdrant 运行状态：**
```bash
curl http://localhost:6333/collections
```

## 4. 一键启动所有数据库（推荐）

### 4.1 创建 `docker-compose.yml`

在项目根目录创建统一的数据库配置文件：

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:16-alpine
    container_name: innocore-postgres
    environment:
      POSTGRES_DB: innocore_ai
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: your_password_here
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  qdrant:
    image: qdrant/qdrant:latest
    container_name: innocore-qdrant
    ports:
      - "6333:6333"
      - "6334:6334"
    volumes:
      - qdrant_data:/qdrant/storage
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:6333/readyz"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: innocore-redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

volumes:
  postgres_data:
  qdrant_data:
  redis_data:
```

### 4.2 启动命令

```bash
# 启动所有数据库
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止所有数据库
docker-compose down

# 停止并删除数据卷（慎用）
docker-compose down -v
```

## 6. 数据库初始化与验证

### 6.1 手动验证连接

```bash
# 测试 PostgreSQL 连接
psql -h localhost -U postgres -d innocore_ai -c "SELECT version();"

# 测试 Qdrant 连接
curl http://localhost:6333/collections

# 测试 Redis 连接
redis-cli -h localhost -p 6379 ping
```

### 6.2 应用启动时自动建表

PostgreSQL 的表结构在 `core/database.py` 中定义，启动应用时会自动执行 `_create_tables()` 创建所需表（使用 `CREATE TABLE IF NOT EXISTS`，安全幂等）。

Qdrant 的集合在 `core/vector_store.py` 中定义，启动应用时会自动调用 `_create_collections()` 创建 `innocore_l1_preset` 和 `innocore_l2_user` 两个集合。

### 6.3 查看应用数据库连接状态

启动应用后，访问健康检查端点：

```bash
curl http://localhost:8000/api/health
```

## 7. 故障排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `Connection refused` | 数据库未启动 | 运行 `docker-compose up -d` |
| `password authentication failed` | 密码错误 | 检查 `.env` 中的 `DATABASE_PASSWORD` |
| `database "innocore_ai" does not exist` | 数据库未创建 | 手动创建：`docker exec -it innocore-postgres psql -U postgres -c "CREATE DATABASE innocore_ai;"` |
| `Qdrant 401 Unauthorized` | API Key 错误 | 本地部署可不设置 API Key |
| `Connection pool exhausted` | 连接数超限 | 检查 `pool_size` 配置，或降低并发请求数 |

## 8. 生产环境建议

1. **使用云数据库服务**（如阿里云RDS、腾讯云PostgreSQL）替代自建
2. **启用SSL/TLS加密**数据库连接
3. **设置强密码**并定期轮换
4. **配置备份策略**（每日自动备份）
5. **监控告警**（CPU、内存、连接数监控）
6. **生产环境移除 `DEBUG=True`**
